Robot Dog · Phase 1

Local Firmware Flasher

This page lets any computer on your network flash a robot dog firmware image from that same computer's browser. Plug the board into the computer you're using, open this page in Chrome or Edge, and hit the install button.

ESP32-C3 · USB flash only · LAN hosted

If the button looks dead, you're probably using Safari. Chrome/Edge is the path of least pain.

How to use it

  1. Build fresh firmware in this repo so build/ contains current artifacts.
  2. Host the repo root on your LAN with a simple static server.
  3. On the target computer, open /tools/local-flasher/index.html in Chrome or Edge.
  4. Plug the robot dog's ESP board into that same computer via USB.
  5. Click Connect, choose the serial port, then install.

What gets flashed

  • 0x0 → bootloader/bootloader.bin
  • 0x8000 → partition_table/partition-table.bin
  • 0xd000 → ota_data_initial.bin
  • 0x10000 → mojbot-firmware.bin
  • 0x2de000 → generated_assets.bin

This manifest is generated from the current ESP-IDF build output, not hand-waved.

Reality check

This does not let one machine directly flash a board physically plugged into another machine. The browser that talks to USB must be running on the computer holding the board.

What this page does buy you: one shared LAN URL, no full dev setup on every laptop, and no one needing to remember raw idf.py flash commands.

Refresh after a new build

From the repo root, run:

python3 scripts/export_web_flasher.py

That regenerates tools/local-flasher/manifest.json from build/flasher_args.json so the page stays aligned with the current firmware layout.
